# Open Engineering Platform (OEP) Foundation

> **Build the Platform. Preserve the Knowledge.**

The Open Engineering Platform (OEP) Foundation is the core platform that powers the OEP ecosystem.

It provides the infrastructure, standards, runtime, repository engine, SDKs, and developer tooling upon which all OEP Studios, Packages, and future engineering applications are built.

This repository contains the reference implementation of the Open Engineering Platform.

---

# Mission

The mission of OEP is to preserve engineering knowledge independently of software applications.

Engineering knowledge should remain accessible, versioned, searchable, and reusable across industries, organizations, and generations.

Applications evolve.

Technologies evolve.

Engineering knowledge should not.

# Engineering Principles

The Open Engineering Platform is built upon the following principles:

- Repository First
- Specifications Before Implementation
- Engineering Objects are the Source of Truth
- Files are Generated Views
- Open Standards
- Commercial Implementations
- Long-Term Stability
- Cross-Platform by Design
- Offline First
- Architecture Before Features

---

# Platform Components

The Foundation currently consists of:

- OEP CLI
- Runtime
- Repository Engine
- Public API
- SDKs
- Exchange Client
- Repository Filesystem

Everything else is built upon these components.

---

# Studios

The Foundation supports multiple engineering-focused applications including:

- Installation Studio
- Service Studio
- Maintenance Studio
- Engineering Studio
- Inspection Studio
- Operations Studio
- Training Studio
- Administration Studio

Studios consume Foundation services through the SDK and Runtime.

# OEP Registry

Engineering knowledge is distributed through the OEP Registry.

The Registry provides:

- Packages
- Studios
- Industry Content
- Templates
- Plugins
- Documentation
- Future AI Integrations

Foundation remains intentionally minimal.

Capabilities are added through packages.
